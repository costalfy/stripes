Examples

Stripes currently ships with these great examples that run from a single WAR file:

  • A simple quickstart example (a simple calculator)
  • A simple ajax version of the quickstart example using regular ActionBeans and Javascript prototype.js
  • A more involved, but still standalone example application, a bug tracker called Bugzooky.
  • New A simple ajax version of the quickstart example using jQuery and RestActionBean!
  • New A Single-Page Application (SPA) using HTML, EmberJS and RestActionBeans. Rock and Roll!
  • New A Single-Page Application (SPA) using HTML, jQuery and RestActionBeans. (Uses JSON request bodies instead of request parameters) To-Do List!
  • New A non-blocking Action Bean (Servlet3) sample

For those of you too young or too old or who did not grow up with western cartoons as a kid, Godzooky (alternative spelling: Godzookie) was the diminutive offspring of Godzilla in the 1970/80s Godzilla cartoons by Hanna Barbara. The name from the example application comes from the fact that it is a example bug tracking application, and is massively smaller than Bugzilla.

The single-page application (Rock and Roll) is a reference application created by Balint Erdi. The front-end is pure HTML, EmberJS and Handlebars. The back-end is nothing but Stripes RestActionBeans. There is a great blog post "how-to" for the front-end Rock and Roll application here. The Github source code for the Rock and Roll application is here.

Stripes Online Resources

More information about Stripes is available online. The following documentation is available to help you learn and develop with Stripes:

  • The Stripes documentation website includes lots of reference and How-To documentation.
  • The Stripes API documentation (JavaDoc)
  • The Stripes issue tracker (JIRA)
  • The Stripes Tag Library documentation (TLDDoc)

If you are developing an application using Stripes you are strongly encouraged to join the Stripes Users' mailing list for support. In addition, confirmed bugs (preferably with test cases) can be reported using Stripes' JIRA bug tracking system.
